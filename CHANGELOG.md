### Decided

- **ADR-0003 — Save storage & versioning is ACCEPTED (M1 · T10)**
  (`design/decisions/0003-save-storage-versioning.md`): saves are **local-first and
  client-owned** — the dependency-free engine keeps doing zero I/O and only emits/consumes the
  T7 `SaveFile` string, which each client persists locally (harness/native → an atomically
  written file under the OS app-data dir; web → **IndexedDB**, not `localStorage`). Persistence
  is a **single rolling autosave slot per run**, written write-temp-then-atomic-rename with a
  retained `.bak`, so no crash can corrupt more than the most recent turn (zero-corruption
  target, NFR-SAVE-02). A save is exportable/importable as text so it doubles as a bug repro
  (DESIGN §9). Migration is the **T7 forward-only ladder** — one integer `SAVE_SCHEMA_VERSION`
  bumped only on a breaking state-shape change, one pure/total `N → N+1` rung per bump,
  newer-than-build refused. Optional cloud sync is deferred post-launch as an additive transport
  over the same string. Unblocks T11+ to autosave real runs toward the M1 lossless quit/resume
  gate.

### Added

- **Six independently-tickable simulation layers — the M2 substrate (M2 · T23 · FR-SIM-01)** (`prototype/engine/src/sim/worldSim.ts`): FR-SIM-01's six state layers already existed in `GameState`; what M1 lacked was the ability to advance the *world* side on its own clock. T23 adds a first-class **world simulation** of six named, pure, independently-tickable layers (`zombies, regions, weather, timeOfDay, hordes, director`) with a uniform `tick(state, ctx)`, a `tickWorld` that folds them in canonical order, and an **`advanceWorld(state, hours, graph?)`** that moves the tracked world slices forward with **no player action** — deterministically and save-losslessly. The fixed 14-stage pipeline is untouched in shape: each world stage's body now delegates to its layer (stage 7 → regions, stage 6 → noise+zombies, stage 8 → weather+timeOfDay, stage 9 → hordes, stage 11 → director), so the invariant order assertion stays green and behaviour was byte-identical the moment it landed. 11 engine tests (layer independence, the six-in-order registry, off-screen advance, determinism, save round-trip, zero-hour inertness).

- **Off-screen regional drift — a region festers while you're elsewhere (M2 · T24 · FR-SIM-03)** (`prototype/engine/src/sim/regionDrift.ts`): the `regions` layer now evolves `threat` and `zombieDensity` every tick before the T17 loot contest. Density relaxes toward a carrying capacity set by the region's own `threat` (breeds) and `survivorActivity` (culls); threat tracks that density and any active fire and bleeds off as the dead disperse — a coupled, integer-clamped relaxation with a tiny per-region `region`-stream jitter so no two regions move in lockstep. It can only ever move a value *toward* a clamped 0–100 target, so it never manufactures an impossible state. **The DoD holds: a region's threat measurably changes across days with the player absent** (verified 35→6 across a live run driven only by `advanceWorld`). No `SAVE_SCHEMA_VERSION` bump — the fields already existed. 9 tests.

- **Zombie state machine + first distinct types (M2 · T25 · FR-CBT-06/07)** (`prototype/engine/src/sim/zombies.ts`): the walkers loitering at a node are now **simulated agents with states** — dormant · wandering · investigating · chasing · feeding · hibernating — that snap up an arousal ladder on senses (the T14 node noise, the player's presence, a bleeding player's scent, the phase) and relax one rung per quiet tick, diverting onto corpses to *feed*.
