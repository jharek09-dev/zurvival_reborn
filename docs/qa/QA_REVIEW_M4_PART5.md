# QA Review — M4 Part 5 (T49): Infection as staged identity

**Scope:** T49 — infection promoted from the T22 *countdown-to-instant-death* into a **staged identity** read only from symptoms: FR-INJ-05 (staged, no bar), FR-INJ-06 (perception / dialogue / symptoms by stage), FR-INJ-07 (diagnosis / cure / quarantine, late costlier & less certain), FR-INJ-08 (no instant Game Over — a harder way to keep playing). Retires the PRD §10 "infection-as-identity is confusing" tripwire.
**Build verified (clean sandbox):** engine **449** · content-loader **9** · harness **85** · typecheck clean ×3 · schema gate **128 entries / 9 types** · malformed rejected · empty-turn smoke exit 0.
**Save schema:** **one rung** — `SAVE_SCHEMA_VERSION 8 → 9` (`migrateV8toV9` re-derives `infection.stage` from `progression` under the new bands). Pipeline: 14-stage order unchanged.
**Owner decision (the crux):** reaching **terminal no longer ends the run** — it opens the cure race; death by infection is a **delayed `succumb`** reached by neglecting that race (the authored heightened death scene stays T62). If the owner prefers terminal to *never* auto-end the run, it is a one-constant change (`INFECT_SUCCUMB_AT = ∞`); this part ships the delayed-succumb reading as the more defensible one (it satisfies both the FR-INJ-08 Must and the GDD's "some runs still end in death by infection").

## What shipped

The M1 loop modelled infection as a death timer: an untreated bite added `+2/h` to a hidden `progression`, and at 100 `runEndReason` returned `"infection"` and the run ended on the spot. GDD Part VI is explicit that this is exactly wrong — *"infection is identity, not a countdown."* T49 rebuilds it in a new `sim/infection.ts` (authoritative dials) with `survival.ts` delegating the driver and run-end:

- **FR-INJ-05 — staged, no bar.** `none → incubating (asymptomatic) → symptomatic → advanced → terminal`. `incubating` is kept as the key for the GDD's *asymptomatic* stage; only `advanced` is inserted, at progression **70**, so the T22 `symptomatic` (40) / `terminal` (100) thresholds are unmoved and every prior `stageFor` assertion holds. The number is never surfaced (FR-UI-02).
- **FR-INJ-08 — no instant Game Over.** `runEndReason` returns `"infection"` **only** at `hasSuccumbed` (`progression ≥ INFECT_SUCCUMB_AT = 148`) — ~24 in-game hours of continued *untreated* driver past terminal onset (100). Between the two is the **cure race**: turns to diagnose, cure, or at least close the bite and live on terminal. Progression runs 0–148 on its own `clampInfection`.
- **FR-INJ-06 — perception, symptoms, dialogue.** The honest **symptom** channel (four distinct, escalating reads in the always-shown status region) is what the comprehension gate pins. The **perception distortion** — hallucinated leads at advanced+, plus memory gaps at terminal — is injected into `sceneOf` by a **stateless hash** (`hashUnit` over `cyrb128`) of `${seed}:${turn}`, so a render is pure, advances no RNG, and is resume-safe. It is framed as possibly-unreal, **suppressed whenever a real danger lead is present**, and never fabricates a real threat (decisions key off real state). NPCs react to the **visible sign**; a **visibly-dying (advanced+) player can no longer recruit** (`encounters.ts`). Stage-scaled **fever fatigue** (1/2/3 per hour) is infection's felt consequence.
- **FR-INJ-07 — diagnosis, cure, quarantine.** `infectionChoices` (mirroring `shelterChoices`) offers **diagnose** (names the stage precisely; the opt-in clarity valve), **treat-infection** (antibiotics — strong & certain early; **stage-scaled and seeded-uncertain** at advanced/terminal via a new lazily-seeded **`infection` RNG stream**), and **quarantine** (shelter clean-conditions — strong early, **not offered at terminal**). Each emits an honest, no-number outcome (*cleared / eased / held*). All three gate on **symptomatic+** so the asymptomatic clock is never leaked by a self-care option; the asymptomatic lever is the ordinary bite `treat`.

**Content — a new schema-validated type.** `content/infections/infection.bite.json` + `content/schemas/infection.schema.json` take the gate **8 → 9 types**. The engine holds the authoritative dials and its own fallback prose (as it already does for weather/atmosphere); the content is the canonical mirror, and a **harness drift-guard** asserts the two stay byte-identical (stage keys/order, symptom, sign, cure amount/partial/certainty, quarantine) — the T40 arc pattern.

**Save schema — one rung, `v8 → v9`.** The stage *bands* changed (an `advanced` band at 70), so a v8 mid-infection save with progression in [70, 100) stored `"symptomatic"` under the old bands. `migrateV8toV9` re-derives `stage` from the stored `progression` for the player and every actor — a pure, total *value normalization*, not a reshape; healthy saves (progression 0 ⇒ `none`) are untouched.

## Verification

- **Every prior bite-free golden is byte-identical.** All infection paths are inert at stage `none`: `updateCondition`, `perceptionDistortion`, `peopleLine`/`infectionSign`, `encounterPeople`, the history observer, `infectionChoices`, and the `sceneOf` additions each verified to no-op when healthy. The engine baseline that was 418 before T49 stayed green; the 445→449 growth is all new tests. The `infection` RNG stream is drawn **only** by an advanced/terminal cure, so no bite-free run ever touches it.
- **FR-INJ-08 proven.** `runEndReason(terminal onset)` is `null` with a non-empty `availableActions`; the run ends only at succumb. A full arc test drives an ignored bite to terminal, plays *more than one* turn there, and then succumbs with reason `"infection"`. The two T22 tests that asserted terminal ⇒ run-over were **inverted** to assert the new behaviour.
- **Determinism.** `perceptionDistortion` is a pure function of state — `sceneOf` twice on one state is byte-identical narration, and no stream advances. The late cure is seeded (same seed ⇒ same full/partial result) and advances **only** the `infection` stream (every other stream byte-identical); the early cure takes no draw at all.
- **Comprehension gate (the tripwire retirement).** For each stage the status carries a **distinct, escalating** symptom keyword; the hidden number never appears in status, scene, *or* the diagnosis line; the honest "can't trust your senses / what is real" frame rides both advanced and terminal (so distrusting the hallucination channel is fair); and a cure at symptomatic drops the stage and **removes the fever line** (relief is felt, no number). An actionable response is on the menu the moment symptoms show.
- **Save-losslessness.** A mid-terminal infection round-trips deep-equal; the v8→v9 rung is covered (an 85-progression `"symptomatic"` v8 save loads as `"advanced"`, number preserved; consistent saves untouched).

## Adversarial audit → fixes applied

Two subagents reviewed in parallel — an engineering lens (correctness / determinism / save / edge cases) and a design/comprehension lens (legibility / no-number / real-trade / voice / fairness). Both confirmed the core FR-INJ-08 mechanic, bite-free determinism, hallucination fairness, and the reliable/unreliable channel split as **sound**. Fixes applied:

- **HIGH — quarantine-at-terminal was an unlabeled dead option** that burned 8h and let the bite drive on for zero benefit. `canQuarantine` now returns false where clean conditions can't act (terminal); the menu simply shrinks (legible, not a leak). The cure, which can still help, remains offered.
- **MED — the new `advanced` stage was missing from the director's `playerDistressed` predicate**, so the world could escalate at advanced while easing at symptomatic *and* terminal — a difficulty dip at the middle stage. Now uses the module's own `isSymptomatic` (symptomatic/advanced/terminal), keeping the curve monotonic.
- **MED — quarantine forfeited the shelter rest bonus:** `updateCondition` treated `quarantine` as a rest but `applyShelterRest` fired only for `"rest"`, so quarantining at your own base left you *more* tired than resting there. `applyShelterRest` now includes `quarantine`.
- **MED — a hallucinated "horde" could collide with a real horde lead.** Perception distortion is now suppressed on any turn a real danger lead is on the board, so it colours only the quiet and can never undermine a genuine threat.
- **MED — terminal dropped the "can't trust your senses" fairness frame** at the most-distorted stage. The terminal symptom now carries "you can no longer tell what is real," so the meta-warning holds right through terminal.
- **MED — a scarce dose gave no feedback on a partial/failed late cure.** Cure and quarantine now log an `infection.treated` beat and `sceneOf` surfaces an honest, no-number outcome line (*cleared / eased / held*) the turn it happens.
- **MED — the comprehension gate under-tested its own guarantee.** Strengthened to assert distinctness + escalation, no-number over status **and** scene **and** the diagnosis line, and that a cure removes the fever line.
- **LOW (voice) — the diagnosis line phrasing** ("it is symptomatic" → "it has reached the … stage") and one purple hallucination line ("the shadows breathe" → a grounded fever image) were reworded.

## Parking lot / deferrals

- **PL-M4-21 — infection sources are bite-only.** The staged machine is source-agnostic (the schema's `source` enum already lists wound / tainted-water / spoiled-food / blood); the non-bite sources arrive with the T51 economy. Latent: the driver keys on the exact wound id `wound.bite` (combat inflicts exactly that today; a future subtyped bite id would need a `WOUND_EFFECTS` entry).

Diagnose is near-free (holds a med item, 1h, no consume) and the plan's clinic-node diagnosis path was cut for a graph-free gate.
